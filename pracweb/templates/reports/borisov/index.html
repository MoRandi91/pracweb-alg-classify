Схема работы


Пользователь вводит настройки модели и описание набора объектов. Может сразу наблюдать визуализацию введенного набора объектов. По запросу пользователя, строится описание задачи в машиночитаемом виде, которое отправляется посредством AJAX-запроса на HTTP-сервер. HTTP-сервер передает описание на обработку серверу задач; отвечает на запросы о статусе (возможно, длительной) операции. Сервер задач берет задачи из очереди задач, проводит полный цикл обработки задачи (валидация описания, построение модели, тестирование модели на контрольном наборе данных, построение двухмерной карты оценок классификации).

Клиент

Представляет собой интерактивное WEB-приложение, написанное с использованием HTML5, CSS3, SVG и JavaScript. В том числе, были использованы JavaScript-библиотеки: jQuery для манипуляции представлением интерфейса, D3.js для реализации отображений между данными и представлением, MathJax для поддержки математических текстов в TeX-нотации в отчетах.

HTTP-сервер

Является тонкой прослойкой между клиентским интерфейсом и сервером задач. Решает две задачи: во-первых, собирает представление интерфейса из шаблонов и отдает HTML-страницы клиенту; во-вторых, реализует AJAX-запросы для передачи данных между клиентом и сервером задач. Реализован на языке Python с использованием библиотеки Flask.

Сервер задач

Обрабатывает долгие операции по обработке задач. Реализован на языке Python, использует библиотеку Celery для диспетчеризации задач и управления очередью задач. {% endblock report %}
